# SYRIX — Functional Specification (المواصفات الوظيفية)

## 1. نظرة عامة
نظام ويب متكامل (Web App) لأكثر من 100 موظف، متعدد الإدارات، يعمل Responsive على موبايل/تابلت/ديسكتوب. المستخدمون: موظفون بصلاحيات محددة حسب إدارتهم + Admin عام يشوف كل شيء.

## 2. نظام الصلاحيات (RBAC)

### الأدوار الأساسية
| الدور | الوصول |
|------|--------|
| Super Admin | كل الإدارات + إعدادات النظام + إدارة المستخدمين |
| Department Manager | إدارته بالكامل (قراءة/كتابة) + تقارير إدارته |
| Employee | شاشاته الشخصية فقط (حضوره، إجازاته، مهامه) داخل إدارته |
| Accountant | موديول الحسابات فقط، بمستويات (محاسب / مراجع / مدير حسابات) |

### مبدأ الصلاحيات
- كل مستخدم يشوف بيانات إدارته فقط، إلا Super Admin.
- كل صلاحية تُفحص في الـ Backend (middleware) قبل أي عملية CRUD — راجع `01-constitution.md` بند 1.

## 3. الوحدات (Modules)

### 3.1 الموارد البشرية (HR)
- ملفات الموظفين (بيانات أساسية، مسمى وظيفي، إدارة، تاريخ التعيين).
- الحضور والانصراف (Check-in/Check-out، تقرير شهري لكل موظف).
- الإجازات (طلب، موافقة مدير، رصيد إجازات متبقي).
- المرتبات (مرتب أساسي، بدلات، خصومات، صافي الراتب) — **مرتبط بموديول الحسابات** (كل مرتب يُصرف = قيد محاسبي).

### 3.2 العمليات
- إدارة الوحدات العقارية (أو الأصول التشغيلية حسب طبيعة النشاط).
- الحجوزات ومتابعة حالتها.
- متابعة التعاقدات (تاريخ البداية/النهاية، تنبيهات انتهاء).

### 3.3 الحسابات (Accounts) — الموديول الأكثر تفصيلاً

> هذا الموديول مبني مباشرة على ملف `Accounting_System_Overview.md` المرفق، ومُترجم هنا لمتطلبات وظيفية قابلة للتنفيذ.

#### أ) دليل الحسابات (Chart of Accounts)
يجب أن يدعم النظام كحد أدنى الحسابات التالية، مصنّفة حسب طبيعتها:

- **مصروفات:** إيجار، مرتبات، مرافق (كهرباء/مياه)، إعلانات، مستلزمات مكتبية، إهلاك، تأمين، فوائد مدينة، مصاريف بنكية، صيانة، مصاريف عمومية، سفر، ديون معدومة، مصاريف متنوعة.
- **إيرادات:** إيراد مبيعات، إيراد خدمات، إيراد فوائد، إيرادات أخرى.
- **أصول متداولة:** نقدية، بنك، عملاء (ذمم مدينة)، مخزون، مصروفات مقدمة.
- **أصول ثابتة:** أراضي، مباني، آلات، سيارات، أثاث، أجهزة كمبيوتر.
- **أصول غير ملموسة:** شهرة، براءات اختراع، برمجيات، علامات تجارية.
- **التزامات:** متداولة (موردون، مصروفات مستحقة) وطويلة الأجل (قروض).
- **حقوق ملكية:** رأس المال، رأس مال إضافي، حساب جاري الشريك، أرباح مرحّلة، أرباح/خسائر العام الحالي، مسحوبات.

#### ب) القيد المزدوج (Double-Entry Engine)
- كل عملية = قيد له طرف Debit وطرف Credit، ومجموعهما متساوٍ إجباريًا (يُرفض القيد لو غير متوازن).
- قاعدة الأرصدة الأمريكية (American Rule) تُطبَّق تلقائيًا في الكود:
  - الأصول والمصروفات تزيد بالمدين (Debit) وتقل بالدائن (Credit).
  - الالتزامات وحقوق الملكية والإيرادات تزيد بالدائن (Credit) وتقل بالمدين (Debit).
- مثال إجباري التنفيذ: قيد الإهلاك الشهري التلقائي:
  `Dr. Depreciation Expense / Cr. Accumulated Depreciation`

#### ج) ميزان المراجعة (Trial Balance)
- تقرير يسرد كل الحسابات بأرصدتها المدينة/الدائنة.
- تحقق تلقائي: إجمالي المدين = إجمالي الدائن (وإلا تظهر رسالة خطأ توازن).

#### د) قائمة الدخل (Income Statement)
تُبنى تلقائيًا بالترتيب:
`إيراد المبيعات − تكلفة البضاعة المباعة = مجمل الربح − المصروفات التشغيلية = الربح التشغيلي − مصروف الفوائد = صافي الربح`

#### هـ) الميزانية العمومية (Balance Sheet)
`الأصول = الالتزامات + حقوق الملكية`
تُقسَّم الأصول لمتداولة/ثابتة، والالتزامات لمتداولة/طويلة الأجل.

#### و) حقوق الملكية
معادلة إلزامية:
`رأس المال الختامي = رأس المال الافتتاحي + صافي الربح + استثمارات إضافية − المسحوبات`

#### ز) قائمة التدفقات النقدية (Cash Flow Statement)
أنشطة تشغيلية + استثمارية + تمويلية → صافي التغير في النقدية → النقدية أول المدة → النقدية آخر المدة.

#### ح) النسب المالية (يُحسَب تلقائيًا من البيانات الفعلية، ويُعرض في Dashboard خاص بالحسابات)
- **سيولة:** Current Ratio، Quick Ratio، Cash Ratio
- **ربحية:** Gross Profit Margin، Net Profit Margin، ROA، ROE
- **ملاءة:** Debt to Assets، Debt to Equity، Times Interest Earned
- **نشاط:** Inventory Turnover، Accounts Receivable Turnover، Total Asset Turnover

#### ط) الفواتير والمصروفات
- إصدار فواتير للعملاء (مرتبطة بموديول المبيعات).
- تسجيل مصروفات وربطها بقيود تلقائية.
- تقارير مالية دورية (شهري/ربع سنوي/سنوي) قابلة للتصدير PDF/Excel.

### 3.4 التسويق
- إدارة الحملات الإعلانية (اسم، ميزانية، تاريخ، قناة).
- إدارة الـ Leads (بيانات عميل محتمل، مصدره، حالته).
- تتبع مصدر العميل (UTM أو إدخال يدوي) وربطه بموديول المبيعات.

### 3.5 المبيعات
- خط سير العميل (Pipeline): مراحل (عميل محتمل → تفاوض → تم البيع → مرفوض).
- متابعات (تذكيرات، ملاحظات على كل عميل).
- تقارير مبيعات (حسب الموظف، الفترة، المصدر) — تُغذّي موديول الحسابات (كل عملية بيع = إيراد).

## 4. متطلبات التوافق مع الأجهزة (Responsive)
- Mobile-First مع Media Queries.
- `<meta name="viewport" content="width=device-width, initial-scale=1">` في كل صفحة.
- اختبار إلزامي على 360px / 768px / 1200px+.
- أزرار بحجم لمس ≥ 44px.
- الجداول: تمرير أفقي على الموبايل، أو تتحول لبطاقات (Cards) — القرار يُترك لكل جدول حسب عدد الأعمدة (أكثر من 5 أعمدة → Cards).

## 5. إدارة المخاطر (Risk Management) — كمتطلب وظيفي عام
- سجل مخاطر (Risk Register) داخل النظام: تحديد الخطر، تقييمه (احتمالية × تأثير)، إجراء التخفيف، تاريخ المراجعة القادمة.
- يظهر كـ Dashboard في موديول الحسابات/الإدارة العليا فقط (Super Admin + Department Manager).
